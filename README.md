# Анидл — угадай аниме по комментариям

Ежедневная браузерная игра: сайт загадывает аниме из топа Shikimori, а вы угадываете его по русским комментариям пользователей за 6 попыток. Каждая ошибка открывает новый комментарий и подсказку (год, формат, жанры, студия, оценка). Есть бесконечный режим. Названия аниме в комментариях автоматически маскируются (▮▮▮), спойлеры вырезаются.

Сайт полностью статический: GitHub Actions раз в неделю собирает данные с Shikimori API и кладёт их в репозиторий, а GitHub Pages бесплатно раздаёт сайт. Сервер не нужен.

## Как опубликовать (5 шагов, всё бесплатно)

1. **Создайте репозиторий.** На github.com нажмите «New repository», имя например `anidle`, тип **Public** (Pages бесплатен только для публичных). Загрузите туда все файлы этого проекта («Add file → Upload files»), сохранив структуру папок. Если загружаете через веб-интерфейс, папку `.github` проще добавить через «Add file → Create new file» с именем `.github/workflows/update-data.yml`, вставив содержимое файла.

2. **Разрешите Actions писать в репозиторий.** Settings → Actions → General → внизу «Workflow permissions» → выберите **Read and write permissions** → Save.

3. **Соберите данные.** Вкладка **Actions** → слева «Обновить данные с Shikimori» → кнопка **Run workflow**. Сбор занимает 10–20 минут (скрипт вежливо соблюдает лимиты Shikimori API). После успеха в папке `data/` появятся настоящие аниме и комментарии вместо демо-заглушек. Дальше данные будут обновляться сами каждый понедельник.

4. **Включите Pages.** Settings → Pages → «Build and deployment» → Source: **Deploy from a branch** → Branch: `main`, папка `/ (root)` → Save.

5. **Готово.** Через пару минут сайт будет доступен по адресу `https://ВАШ_ЛОГИН.github.io/anidle/`. Новая загадка выбирается автоматически каждый день в 00:00 МСК — ничего запускать не нужно.

## Структура

```
index.html                          страница игры
css/style.css                       оформление
js/app.js                           логика игры
data/animes.json                    пул аниме (русские названия, подсказки)
data/comments/{id}.json             комментарии по каждому аниме
scripts/fetch-data.mjs              сборщик данных с Shikimori API
.github/workflows/update-data.yml   автообновление данных (Actions)
```

## Настройка

В начале `scripts/fetch-data.mjs`:
- `POOL_TARGET` — размер пула аниме (по умолчанию 220);
- `MIN_COMMENTS` / `MAX_COMMENTS` — сколько комментариев нужно/сохраняется;
- фильтр популярности и типов — в функции `collectPool`.

В `js/app.js`: `MAX_TRIES` — число попыток.

## Примечания

- Данные и комментарии принадлежат Shikimori и его пользователям; на сайте стоит ссылка на источник.
- Прогресс и статистика игрока хранятся в localStorage браузера.
- Ответ дня технически можно найти в коде страницы — как и во всех играх такого типа (Wordle и т. п.). Для домашнего проекта это нормально.
